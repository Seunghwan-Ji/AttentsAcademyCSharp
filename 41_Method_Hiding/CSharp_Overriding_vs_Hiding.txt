C# Method Overriding vs Method Hiding

1. Method Overriding (재정의)

조건

-   부모 메서드에 virtual 필요
-   자식 메서드에 override 사용

특징

-   실제 객체 타입 기준으로 실행됨
-   다형성(Polymorphism) 지원

예제

class Animal { public virtual void Speak() { Console.WriteLine(“Animal
sound”); } }

class Dog : Animal { public override void Speak() {
Console.WriteLine(“Bark”); } }

Animal a = new Dog(); a.Speak(); // 출력: Bark

2. Method Hiding (메서드 숨김)

조건

-   부모 메서드에 virtual 없어도 됨
-   자식 메서드에 new 사용

특징

-   변수 타입 기준으로 실행됨
-   다형성 아님

예제

class Animal { public void Speak() { Console.WriteLine(“Animal sound”);
} }

class Dog : Animal { public new void Speak() {
Console.WriteLine(“Bark”); } }

Animal a = new Dog(); a.Speak(); // 출력: Animal sound

핵심 비교

Overriding: - override 키워드 - 부모 virtual 필요 - 실제 객체 기준 -
다형성 O

Hiding: - new 키워드 - 부모 virtual 불필요 - 변수 타입 기준 - 다형성 X

한 줄 정리

Overriding = 부모 기능을 재정의 (진짜 교체) Hiding = 이름만 같은 새
메서드 (가려짐)
